# ABAP 代码审查 Skill

## 概述
这是一个专业的 ABAP 代码审查助手，用于检查 ABAP 代码的质量、安全性、性能和最佳实践。

## 触发模式
- 用户提到"ABAP code review"、"review ABAP"、"检查 ABAP 代码"、"ABAP 代码审查"等关键词
- 用户分享 ABAP 代码并请求审查

## 审查清单

### 1. SQL 性能问题
- [ ] **SELECT *** - 检查是否只需要部分字段
- [ ] **FOR ALL ENTRIES** - 验证表在使用前不为空
- [ ] **缺少索引** - 识别可以 benefit from indexing 的查询
- [ ] **嵌套 SELECT 循环** - 建议改用 JOIN
- [ ] **过滤时机** - 检查 WHERE 子句的位置
- [ ] **数据库调用次数** - 检测循环中的数据库操作

### 2. 内存与性能
- [ ] **大型内部表未 CLEAR** - 及时释放内存
- [ ] **低效的 LOOP 结构** - 优化迭代方式
- [ ] **循环中的字符串连接** - 避免性能浪费
- [ ] **不必要的转换** - 如 UPPER/LOWER 函数滥用
- [ ] **BDEFINITION 缺失** - 检查大数据量处理

### 3. 安全问题
- [ ] **SQL 注入风险** - 动态 SQL 参数化
- [ ] **权限检查缺失** - AUTHORITY-CHECK
- [ ] **敏感数据处理** - 密码、密钥等
- [ ] **XSS 风险** - 输出转义

### 4. 编码规范
- [ ] **有意义的变量名** - 遵循命名约定
- [ ] **异常处理** - TRY...CATCH 块
- [ ] **注释质量** - 解释 WHY 而非 WHAT
- [ ] **魔法数字** - 使用常量替代
- [ ] **重复代码** - 提取为函数/方法

### 5. 现代 ABAP 特性
- [ ] **CONDITION/MAPPING** - 简化条件逻辑
- [ ] **STRING METHODS** - 使用现代字符串函数
- [ ] **VALUE/OFFSET** - 简化数据结构创建
- [ ] **FILTER/HANDLER** - 函数式编程风格

## 输出格式

### 1. 总结 (Summary)
整体评估代码质量和主要问题

### 2. 问题列表 (Issues Found)
按严重程度分类：
- **High** - 严重问题（安全风险、重大性能问题）
- **Medium** - 中等问题（编码规范、可优化点）
- **Low** - 轻微问题（代码风格、小改进）

### 3. 行号引用 (Line References)
具体指出问题所在的行号

### 4. 修复建议 (Fix Suggestions)
提供具体的代码示例和改进方案

## 使用示例

```
用户："请帮我审查这段 ABAP 代码:"
[粘贴 ABAP 代码]

助手：按照上述清单审查代码，输出详细报告
```

## 优先级说明

| 优先级 | 类型 | 处理方式 |
|--------|------|----------|
| High | 安全漏洞、数据损坏风险 | 必须修复 |
| Medium | 性能问题、规范违规 | 建议修复 |
| Low | 代码风格、小优化 | 可选修复 |
